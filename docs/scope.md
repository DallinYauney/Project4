The scope of this project mostly conforms to the requirements of the project - track tile landings for a CPU player - but some of the decisions we've made have probably been influenced by the broader functionality of Monopoly. For example, we have a player object and a board object, with the functionality split between them corresponding to how we would actually play Monopoly. This didn't have to happen, we could have combined both of them into one monolith (we only have one player, after all) but the structure and scope are, I think, influenced by the way we think about the project.

The required features are, to paraphrase the Canvas page, to make a program with both the rules of Monopoly and the logic to simulate millions of turns of CPU on that game, with the exceptions that we don't need to simulate money in any way, and we only have to worry about one CPU player with two very simple strategies. This means it's mostly down to board mechanics like movement and speeding, and the decks of cards.

I'm also factoring into the scope my secret hope that, once the game is done, I can port it to a game engine like Godot and turn it into a full game of Monopoly, but I think there's too much extra functionality for this to be feasible in one or two afternoons (my threshold for whether or not I'll finish something this boring).
